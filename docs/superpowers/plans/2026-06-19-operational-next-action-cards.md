# Operational Next-Action Cards Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a first batch of source-backed, first-aid-style operational cards that answer "what happens next?" during common Sivilforsvaret workflows.

**Architecture:** Use the existing `content/curated/action-cards.yaml` action-card model. Each card is compact on the surface, uses richer step objects for immediate action plus optional `how`, links to approved public source IDs, and is marked `pending-fagperson` where source material is synthesized into operational guidance.

**Tech Stack:** Next.js content pipeline, YAML curated content, Zod validation, Vitest fixture tests, MiniSearch generated index.
